# 半调网屏与印刷效果生成器学习路径

## 项目定位

本项目定位为：**一个 local-first、纯前端、本地图像处理的轻量印刷效果编辑器**。

目标不是做一个单一的半调实验器，而是做一个**类似 PS 的少功能简化版**，专注于：
- 半调网屏效果
- 基础印刷风格效果
- 本地实时预览
- 本地导出结果

项目内只保留两条主线，且两条主线都需要完整支持：

1. **预设后细调**
   - 先选择一个预设效果
   - 再继续修改参数
   - 适合快速出结果

2. **从头搭建效果**
   - 不依赖预设
   - 用户自己从空白参数开始组合效果
   - 适合有明确想法、希望自主控制效果的人

这两条主线不是附属关系，而是本项目产品设计的核心。

## 3. 图像处理基础：为了做“类似 PS 简化版”的前置调整

这一部分很关键，因为你的产品不只是套一个半调算法，而是要有**基础调图 + 印刷效果生成**的编辑链路。

### 需要学习
- 灰度化
- 亮度计算
- 阈值化
- 对比度调整
- 亮度调整
- 饱和度基础概念
- Gamma 基础
- 模糊与平滑基础概念
- 反相处理

### 需要理解
在这个项目里，用户往往不会直接拿原图去做半调，而是会先做一些基础整理：
- 提高对比度，让网点更清晰
- 调整亮度，让暗部和亮部关系更合理
- 改变阈值，让黑白分界更明确
- 反相，得到不同印刷风格

### 需要达到
- 能做基础调图模块
- 能让这些基础调整作为效果链的一部分参与渲染
- 能支持预设自动填参，也能支持用户从空白参数自己搭建

## 5. 半调网屏核心原理：这是项目最核心的部分

### 需要学习

- FM Halftone 的概念了解
- 亮度到点大小映射
- 网角概念
- 坐标旋转基础
- 半调图形样式：
  - 圆点
  - 方点
  - 线条
  - 椭圆（可选）

### 你必须真正理解
半调不是“滤镜覆盖”，而是：
- 读取亮度
- 按网格重新组织信息
- 用点、方块或线条重新绘制图像

### 第一阶段必须做出的效果
- 黑白圆点半调
- 方点半调
- 线条半调
- 角度控制
- 反相

### 第二阶段可学习
- 双色印刷风格
- 假 CMYK 分层
- 套印偏移感
- 纸张纹理叠加

## 6. 产品结构设计：为了让两条主线真正可用

这是这次调整里最重要的新要求。

### 你需要学习
- 参数系统怎么设计
- 预设系统怎么设计
- 编辑器流程怎么设计
- 模式切换怎么设计
- 设置重置逻辑
- 效果链顺序怎么组织

### 你需要明确的产品结构
本项目只有两条主线：

#### 主线 A：预设后细调
流程：
1. 导入图片
2. 选择预设效果
3. 进入参数面板继续修改
4. 预览结果
5. 导出

#### 主线 B：从头搭建效果
流程：
1. 导入图片
2. 不选预设，直接进入编辑器
3. 自己设置基础调整
4. 自己设置半调与印刷参数
5. 预览结果
6. 导出

### 需要达到
- 预设不是终点，而是起点
- 从头搭建不是“高级附属功能”，而是完整主线
- 界面上必须清楚区分并支持这两条路径

## 8. 状态管理与模块拆分：为了让项目不写乱

### 需要学习
- 配置对象组织方式
- 编辑器状态拆分
- 预设状态与参数状态关系
- 当前图片状态
- 当前效果链状态
- 导出设置状态

### 你需要的关键认知
为了支持两条主线，参数不能只写成“预设覆盖一切”的方式。

你应该支持：
- 一套默认空白参数
- 一套预设填充逻辑
- 一套用户后续覆盖逻辑

也就是说：
- 预设可以写入参数
- 用户可以继续改写参数
- 用户也可以从一开始就只用默认参数自行搭建
